## Demo Script (5 minutes)

### Opening (30 sec)

> "This is AgentSwap — a system where two AI agents negotiate a freelance contract,
> lock funds atomically on Bitcoin Lightning AND Ethereum simultaneously, and a third
> AI arbitrator decides whether the work was good enough to release payment.
> No humans are involved. Watch."

### Act 1 — Happy Path: Translation Task (~2 min)

1. In Demo Controller (bottom of center column), click **Translation Task**
2. Point to the phase stepper: *"Two AI agents are negotiating right now — you can see their messages on the left and right columns"*
3. When phase transitions to **LOCKED**: *"Both chains are locked simultaneously. That's the atomic swap guarantee — neither party can run away with the money"*
4. When phase transitions to **EVALUATING**: *"A third Claude instance — the arbitrator — is now reading the contract spec and the translation side by side"*
5. When the verdict appears: *"The arbitrator scores the work out of 100 across four criteria. Above 70 means approved"*
6. When **SETTLED** flashes green: *"The same 32-byte secret that unlocked the Bitcoin Lightning invoice was used to claim the Ethereum HTLC — that's cross-chain atomicity"*

**Point out**: "Humans Involved: 0" counter in the center column.

### Act 2 — Failure Path: Bad Delivery (~1.5 min)

1. Click **Bad Delivery** in Demo Controller (available once Act 1 settles)
2. When arbitrator evaluates: *"This time the seller submitted garbage. The arbitrator doesn't know it's a demo — it's running a real LLM evaluation"*
3. When **REFUNDED**: *"Score came back below 70. Buyer gets refunded on both chains automatically — the preimage is never revealed so the seller can't claim anything"*

### Act 3 — What to Highlight to Judges (30 sec)

> "Three things that are real here: The Claude agents are actually negotiating via back-and-forth API calls.
> The HTLCs are real contracts on a local Ethereum node and real Lightning channels.
> The arbitrator is a separate Claude call with no shared context — it has no idea what answer is 'expected'."

## If Something Goes Wrong

### Swap is stuck (hasn't moved in 60 sec)

Use the **Emergency Overrides** in Demo Controller:
- **Force Settle** — bypasses arbitration, reveals preimage immediately
- **Force Refund** — cancels invoice, refunds both chains

### Negotiation failed / no agreement

Click **Reset Demo**, then re-launch the same scenario. Negotiation has stochastic LLM outputs — occasionally fails to agree; just retry.

### LND invoice timeout

The HODL invoice expires after ~10 minutes. If the demo stalls before BTC locks, reset and restart. Faster fix: switch on **Fast Mode** which tightens all delays.

### Dashboard shows "reconnecting…"

Server crashed. Run `pnpm --filter @agentswap/server dev` again. SSE client reconnects automatically within 3 sec.

### ETH transaction fails (gas / nonce)

Restart ganache: `pnpm docker:down && pnpm docker:up`, then redeploy the contract and update `.env.local`.

### Anthropic rate limit hit

You have one API key — don't run multiple swaps in parallel. Always Reset before starting the next scenario.

## Narrated Mode

Enable **Narrated Mode** in Demo Controller before the demo. Each phase transition triggers an overlay banner at the bottom of the screen with a typewriter effect. The text is written to be read aloud by the presenter — it gives judges context without you having to memorize lines.

Narrated mode and Fast mode are mutually exclusive.

## Common Judge Questions

**Q: Is the arbitration actually running Claude?**
A: Yes — `ArbitratorAgent` makes a real `messages.create` API call on every evaluation. You can see the Anthropic request in the server logs.

**Q: What stops the seller from cheating?**
A: The preimage (random 32-byte secret) is generated by the buyer and held in the `PreimageVault`. The seller's Lightning invoice uses an HTLC hash derived from that secret. Until the arbitrator approves and releases the preimage, the seller cannot settle the invoice or claim the ETH.

**Q: What stops the buyer from cheating?**
A: The ETH is locked in the `HashedTimelockETH` contract before BTC is paid. If the buyer never pays the Lightning invoice, the ETH refunds automatically after the timelock expires. The buyer can't pull ETH back once locked.

**Q: Is this production-ready?**
A: No — it runs on a local regtest Bitcoin network and Ganache. The hash function mismatch (Lightning uses SHA-256, Ethereum uses keccak256) is bridged by the AI arbitrator rather than a relay, which is the novel idea we're demonstrating. A production version would use a hash adapter contract.

**Q: Why AI arbitration instead of smart-contract verification?**
A: Smart contracts can verify simple proofs (hash preimages, signatures), but not "was this translation good?". AI arbitration extends the trustless escrow model to arbitrary off-chain deliverables — that's the new design space this explores.

**Q: How do the agents negotiate?**
A: BuyerAgent sends an initial `SwapProposal` (sats, wei, timelocks) as JSON. SellerAgent receives it, evaluates against its minimum rate, and either accepts, counters, or rejects. Up to 5 rounds. Both agents call `chatJSON()` — a thin Anthropic SDK wrapper that returns structured JSON.
